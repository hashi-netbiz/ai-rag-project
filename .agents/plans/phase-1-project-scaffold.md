# Feature: Phase 1 — Project Scaffold & Config

The following plan should be complete. Validate tool versions and directory state before running any command.

Pay special attention to uv's project init behavior — it creates sample files that must be cleaned up.

## Feature Description

Bootstrap the full project skeleton: Python backend (FastAPI via uv), React TypeScript frontend (CRA), all dependency declarations, environment variable template, and empty placeholder files for every module that will be implemented in later phases. No business logic is written here.

## User Story

As a developer,
I want a fully scaffolded project with all dependencies declared and installable,
So that subsequent phases can implement logic into pre-existing, correctly structured files without worrying about tooling or project layout.

## Problem Statement

The project currently has only documentation and source data. Nothing is runnable. Before any logic can be written, the project structure, package manifests, and environment template must exist.

## Solution Statement

1. Initialize the Python backend with `uv init`, declare all dependencies in `pyproject.toml`, and install them.
2. Create `.env.example` with every required environment variable.
3. Scaffold the React TypeScript frontend with CRA and add Axios.
4. Create all placeholder Python and TypeScript files (with `pass` / empty exports) so the directory tree matches the architecture exactly.

## Feature Metadata

**Feature Type**: New Capability (scaffold)
**Estimated Complexity**: Low
**Primary Systems Affected**: backend/, frontend/
**Dependencies**: uv 0.7+, Node 22, npm 11, Python 3.11+

### Patterns to Follow

**Naming Conventions** (from CLAUDE.md):
- Python files: `snake_case.py`
- Python functions/variables: `snake_case`
- Python classes: `PascalCase`
- TypeScript components: `PascalCase.tsx`
- TypeScript utilities: `camelCase.ts`

**Placeholder Python files**: Use a single `# TODO: implement in later phase` comment — do NOT use `pass` alone in modules (confusing); a comment is clearer.

**Placeholder TypeScript files**: Minimal empty export — e.g., `export {};` for utilities, `export default function Foo() { return null; }` for components. This ensures TypeScript doesn't error on import.

### TASK 2 — UPDATE backend/pyproject.toml

Overwrite the generated `pyproject.toml` with the full dependency list. The final file should look like:

```toml
[project]
name = "rag-rbac-chatbot"
version = "0.1.0"
description = "RAG-based RBAC chatbot for company intranet"
requires-python = ">=3.11"
dependencies = [
    "fastapi>=0.110.0",
    "uvicorn[standard]",
    "langchain>=0.2.0",
    "langchain-groq",
    "langchain-pinecone",
    "langchain-google-genai",
    "google-generativeai",
    "pinecone>=3.0.0",
    "python-jose[cryptography]",
    "passlib[bcrypt]",
    "langsmith",
    "ragas",
    "pandas",
    "python-dotenv",
    "tiktoken",
    "pydantic-settings",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

- **GOTCHA**: The PyPI package is `pinecone` (not `pinecone-client`) for v3.x+. Import in code is still `from pinecone import Pinecone`.
- **GOTCHA**: `pydantic-settings` is required for FastAPI's `BaseSettings` in Pydantic v2 — it's a separate package from `pydantic`.
- **GOTCHA**: `ragas` pulls in many transitive deps; install may be slow.
- **VALIDATE**: File saved and parseable — `cd backend && uv run python -c "print('ok')"` (just checks uv can run)

## NOTES

**pinecone package name**: The PyPI package is `pinecone` (v3+), NOT `pinecone-client`. The PRD says "pinecone-client 3.x" but that name was deprecated. Use `pinecone>=3.0.0` in pyproject.toml.

**pydantic-settings**: FastAPI with Pydantic v2 requires `pydantic-settings` as a separate install for `BaseSettings`. Add it explicitly — it's easy to miss and causes a confusing ImportError in Phase 4.

**CRA deprecation**: Create React App is no longer actively maintained. On Node 22 it may produce deprecation warnings but should still function. If it hard-fails, use Vite (`npm create vite@latest frontend -- --template react-ts`) — the resulting structure is nearly identical and compatible with Phases 5+.

**ragas conflicts**: ragas transitively depends on specific versions of langchain/openai. If `uv sync` fails with conflicts, try `uv add ragas --resolution lowest-direct` or pin `ragas>=0.1.0,<0.2.0`.

**uv.lock**: `uv sync` generates `backend/uv.lock` — this should be committed to git for reproducible installs.
